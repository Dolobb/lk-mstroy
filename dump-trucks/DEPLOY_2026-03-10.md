### Шаг 3: Пересчёт travel times для существующих данных

Travel times вычисляются из zone_events (уже в БД). Нужен пересчёт для каждой даты/смены.

**Вариант А — через админ-панель:**
Открыть http://localhost:5173/admin → «Пересчитать самосвалы» (пересчитывает одну дату+смену за раз).

**Вариант Б — массовый пересчёт всех дат:**
```bash
# Список всех уникальных дат/смен:
/usr/local/opt/postgresql@17/bin/psql -p 5433 -d mstroy -c "
  SELECT DISTINCT report_date::text, shift_type
  FROM dump_trucks.shift_records
  ORDER BY report_date DESC, shift_type
"

# Пересчёт одной даты+смены:
curl --noproxy localhost -X POST \
  "http://localhost:3002/api/dt/admin/recalculate?date=2026-03-09&shift=shift1"

curl --noproxy localhost -X POST \
  "http://localhost:3002/api/dt/admin/recalculate?date=2026-03-09&shift=shift2"
```

**Вариант В — скрипт для всех дат:**
```bash
for row in $(/usr/local/opt/postgresql@17/bin/psql -p 5433 -d mstroy -t -A -c "
  SELECT DISTINCT report_date::text || '|' || shift_type
  FROM dump_trucks.shift_records
  ORDER BY 1
"); do
  date=$(echo "$row" | cut -d'|' -f1)
  shift=$(echo "$row" | cut -d'|' -f2)
  echo "Recalculating $date $shift..."
  curl -s --noproxy localhost -X POST \
    "http://localhost:3002/api/dt/admin/recalculate?date=$date&shift=$shift" | head -c 100
  echo
done
```

### Шаг 4: Обновление заявок (необязательно)

Данные заявок (nameCargo, weightCargo, notes, comment и т.д.) уже находятся в `requests.raw_json` — они были загружены ранее при обычном `fetch`. Никакого пересчёта не нужно — фронтенд просто начинает читать поля, которые раньше игнорировал.

Если заявки устарели, можно обновить:
```bash
curl --noproxy localhost -X POST \
  "http://localhost:3002/api/dt/admin/fetch?date=2026-03-10&shift=shift1"
```
