# Admin — Управление сервисами

## Запуск

```bash
# Из корня монорепо — запускает всё одной командой:
npm run dev
```

Запустятся:
- **Admin-сервер** `:3005` — управляет бэкендами
- **Frontend** `:5173` — сам открываешь в браузере

Admin автоматически поднимает все 5 бэкендов при старте.

Страница управления: **http://localhost:5173/admin** (ссылка «Серверы» в шапке)

---

## Карточки сервисов

| Индикатор | Значение |
|-----------|---------|
| 🟢 Зелёный | Сервис запущен и отвечает на порту |
| 🟡 Жёлтый | Процесс запущен, порт ещё не открыт (стартует) |
| ⚫ Серый | Процесс не запущен |

**Кнопки:**
- **Запустить** — активна только когда сервис остановлен
- **Перезапустить** — всегда активна; убивает процесс и запускает заново через 1.5с
- **Стоп** — активна только когда сервис запущен
- **Логи** — раскрывает терминал с последними 80 строками, обновляется каждые 2с

## Покрытие данных

Показывает какие дни загружены в БД для **КИП техники** и **Самосвалов**.

- Выбери период → нажми **Обновить**
- Зелёная клетка = есть данные за этот день
- Серая = нет данных (день не загружался или pipeline упал)
- Наведи мышь на клетку → всплывает дата

> vehicle-status и Тягачи не показываются: их данные не привязаны к конкретным датам (статус ТС — снапшот; тягачи — заявки по периодам).

## Когда нужен перезапуск вручную?

| Ситуация | Что делать |
|----------|-----------|
| Изменил `.ts` / `.py` файл | **Ничего** — tsx watch / uvicorn --reload сделают сами |
| Изменил React-компонент | **Ничего** — Vite HMR обновит в браузере |
| Изменил `.env` | Нажать **Перезапустить** для нужного сервиса |
| Сделал `npm install` в подпроекте | Нажать **Перезапустить** |
| Запустил миграцию БД | **Ничего** (если сервер уже работает, он подхватит новую схему при следующем запросе) |
| Сервис упал с ошибкой и завис | Нажать **Перезапустить** |

## Возможные проблемы

### Все сервисы серые сразу после запуска
**Норма.** Бэкенды стартуют 3–10 секунд. Жди — индикаторы позеленеют.

### Сервис застрял в жёлтом
Процесс запущен, но порт не открывается. Смотри логи — скорее всего ошибка в коде или БД недоступна.

### «Не удалось подключиться к admin-серверу (:3005)»
Admin-сервер не запущен. Запустить:
```bash
cd admin && npm run dev
```
Или из корня монорепо: `npm run dev`.

### Логи пустые у сервиса
Сервис ещё не запускался с момента старта admin-сервера. Нажми **Запустить**.

### Покрытие данных пустое / не загружается
Admin-сервер не может подключиться к БД. Проверить:
```
admin/.env → KIP_DB_PORT, MAIN_DB_PORT, DB_USER
```
По умолчанию: КИП = PG16 порт 5432, остальные = PG17 порт 5433.

### Тягачи (tyagachi) не запускаются
Python должен быть в PATH. Проверить:
```bash
python --version   # или python3 --version
```
Если команда `python` не найдена — в `admin/server.ts` заменить `'python'` на `'python3'`.

### После перезапуска admin-сервера все бэкенды падают
Это ожидаемо — дочерние процессы привязаны к admin-процессу. При остановке admin все они завершаются. Снова запусти `npm run dev`.
