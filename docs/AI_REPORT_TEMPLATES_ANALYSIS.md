## Лист 1: "Отчёт КИП" (186 строк, 15 столбцов)

### Структура

```
Строки 1-2: ЗАГОЛОВОК (merge C1:N2)
  "Выработка техники на АО Мостострой-11 в период с DD.MM.YYYY по DD.MM.YYYY"
  Шрифт: Arial Narrow, Bold, sz11

Строка 3: пустая

Строки 4-5: ДВУХУРОВНЕВЫЕ ЗАГОЛОВКИ СТОЛБЦОВ
  Уровень 1 (строка 4):
    C: № | D: №п/п | E: Марка/гос.№ | F: Объект строительства
    G-J (merge): "1 смена"
    K-N (merge): "2 смена"
  Уровень 2 (строка 5):
    G: КИП% | H: Время нахождения на объекте | I: Время работы двигателя | J: Работа под нагрузкой%
    K: КИП% | L: Время нахождения на объекте | M: Время работы двигателя | N: Работа под нагрузкой%

Далее: данные с 3-УРОВНЕВОЙ ГРУППИРОВКОЙ:
  → Тип ТС: "Тип Бульдозеры" (merge C:N, строка-разделитель)
    → Подразделение: "ДСУ Мостострой-11" (merge C:N, строка-разделитель)
      → Строки данных: №, №п/п, название ТС, объект, 8 значений (по 4 на смену)
```

### Цветовая схема

| Элемент | Цвет фона | Цвет текста | Шрифт |
|---------|-----------|-------------|-------|
| Заголовок (строки 1-2) | Без фона | Чёрный (theme:1) | Arial Narrow, Bold, sz11 |
| Основные заголовки (№, Марка, Объект) | Тёмно-синий #203764 | Белый #FFFFFF | Arial Narrow, Bold, sz7 |
| "1 смена" подзаголовки (G-J) | Средне-синий #2F5496 | Белый #FFFFFF | Arial Narrow, Bold, sz7 |
| "2 смена" подзаголовки (K-N) | Тёмно-синий #203764 | Белый #FFFFFF | Arial Narrow, Bold, sz7 |
| Тип ТС (группа) | Средне-синий (theme:4, tint:-0.25) | Белый (theme:0) | Arial Narrow, Bold, sz7 |
| Подразделение (подгруппа) | Светло-голубой (theme:4, tint:0.8) | Чёрный (theme:1) | Arial Narrow, Bold, sz7 |
| Данные | Белый (theme:0) | Чёрный (theme:1) | Arial Narrow, sz7; КИП и нагрузка — Bold |

### Особенности форматирования

- **96 объединённых ячеек** (merges)
- Двойная нумерация: глобальный № (C) и внутри подразделения (D)
- КИП хранится как десятичная дробь (0.78), отображается как процент через формат ячейки
- Время хранится как Excel serial date, отображается через формат [ч]:мм
- Если ТС работает только в 1 смену → ячейки K:N объединены и содержат текст "Работа в 1 смену"
- Одна машина на 2 объектах → 2 строки с одинаковым номером, ячейки C+D+E объединены вертикально
- Столбцы A и B — пустые отступы (ширина 3.9 и 2.6)
- Все ячейки: тонкие рамки (thin), выравнивание center/middle, wrapText на заголовках
- Высота строк данных: 22.5; высота заголовков строка 5: 33.75

### Данные из БД

Источник: PostgreSQL `kip_vehicles`, таблица `vehicle_records`

| Столбец шаблона | Поле БД |
|----------------|---------|
| КИП% | `kip_percent` |
| Время на объекте | `onsite_time` |
| Время работы двигателя | `engine_hours` |
| Работа под нагрузкой% | `load_percent` |

Дополнительные поля в БД (можно добавить в отчёт):
- `fuel_consumed` — расход топлива (факт)
- `fuel_norm` — расход топлива (норма)
- `mileage` — пробег

## Лист 2: "Отчёт (самосвал)" (43 строки, 6 столбцов) — СВОДНЫЙ

### Структура

**Две отдельные таблицы на одном листе:**

#### Таблица A (строки 1-14): Выработка по времени

```
Строка 1: ЗАГОЛОВОК (merge A1:F1)
  "Выработка Самосвалов в период DD.MM.YY - DD.MM.YY"
  Шрифт: Calibri, Bold, sz16

Строки 3-4: ДВУХУРОВНЕВЫЕ ЗАГОЛОВКИ
  Строка 3: A(merge 3-4): Марка/гос.№ | B-F (merge): без текста (подзаголовки ниже)
  Строка 4: | B: Время работы двигателя | C: Время в движении | D: Время простоя (хол.ход) | E: Выработка% от двиг. | F: Выработка% от движения

Данные:
  → Группа перевозчика: "Самосвалы ТК СпецТранс" (merge A:F)
    → ТС сводка: "Самосвал VOLVO гос.№ р009уа72" (светло-голубой) + ИТОГО значения
      → Дата 1: 17.06.2023 | значения по столбцам (пунктирные рамки)
      → Дата 2: 18.06.2023 | значения
      → ...
    → Следующее ТС...
```

#### Таблица B (строки 16-43): Рейсы и стоянки

```
Строки 16-18: ТРЁХУРОВНЕВЫЕ ЗАГОЛОВКИ
  Строка 16: A(merge 16-18): Марка/гос.№ | B-F(merge): "Итого"
  Строка 17: | B(merge 17-18): Кол-во рейсов | C-D(merge): Средняя стоянка | E-F(merge): Среднее время в пути
  Строка 18: | | C: Погрузка | D: Выгрузка | E: Погрузка | F: Выгрузка

Данные:
  → Объект: "г. Тобольск" (merge A:F)
    → ТС: "Самосвал Volvo FM Truck 6х4 гос.№ Р030МС72" (светло-голубой)
      → Дата 1: 26.01.2026 | рейсы | стоянки | время в пути
      → ...
```

### Цветовая схема

| Элемент | Цвет фона | Цвет текста | Шрифт |
|---------|-----------|-------------|-------|
| Заголовок (строка 1) | Без фона | Чёрный | Calibri, Bold, sz16 |
| Заголовки столбцов (основные) | Тёмно-синий (theme:4, tint:-0.25) | Белый (theme:0) | Calibri, Bold, sz11/sz8 |
| Подзаголовки столбцов | Ещё темнее (theme:4, tint:-0.5) | Белый (theme:0) | Calibri, Bold, sz8 |
| Группа/объект | Средне-голубой (theme:4, tint:0.4 или 0.6) | Чёрный (theme:1) | Calibri, Bold, sz11-12 |
| ТС (сводка) | Светло-голубой (theme:4, tint:0.8) | Чёрный (theme:1) | Calibri, sz11 |
| Данные (даты) | Без фона | Чёрный | Calibri, sz11 |

### Особенности

- 10 объединённых ячеек
- Рамки: medium для внешних границ групп, dashed+thin для строк-дат
- Высота строк заголовков: 15-57.75; данных: 15.75
- Таблица B — шаблон (данные пустые, только структура)
- Шрифт на данных Times New Roman sz10 в столбце B таблицы B (непоследовательно)

### Данные из БД

Источник: PostgreSQL `mstroy`, схема `dump_trucks`

**Таблица A:**
| Столбец | Источник |
|---------|---------|
| Время работы двигателя | `shift_records.engine_duration` |
| Время в движении | `shift_records.moving_duration` |
| Простой (хол. ход) | `engine_duration - moving_duration` (вычисляемое) |
| Выработка% от двигателя | `shift_records.kip_percent` |
| Выработка% от движения | `shift_records.moving_percent` |

**Таблица B:**
| Столбец | Источник |
|---------|---------|
| Кол-во рейсов | `shift_records.trips_count` или COUNT из `trips` |
| Средняя стоянка погрузка | AVG(`trips.loading_duration`) |
| Средняя стоянка выгрузка | AVG(`trips.unloading_duration`) |
| Среднее время в пути П→В | AVG(`trips.travel_to_unload`) |
| Среднее время в пути В→П | AVG(`trips.travel_to_load`) |

## Лист 3: "Отчет по рейсам самосвалов" (199 строк, 16 столбцов) — ДЕТАЛЬНЫЙ

### Структура

```
Строка 1: ЗАГОЛОВОК ДАТЫ+СМЕНЫ (merge A1:P1)
  "21.03 — Cмена 1"
  Фон: серый #D9D9D9, шрифт: Calibri, Bold, sz12

Строки 2-3: ДВУХУРОВНЕВЫЕ ЗАГОЛОВКИ (merge по группам)
  Строка 2: A: № | B: ГосНомер | C: Кол-во рейсов | D: Начало смены
             E-G(merge): "Погрузка" | H-J(merge): "Выгрузка"
             K: Конец смены | L: Средняя стоянка П | M: Средняя стоянка В
             N: Ср. путь Погр-Выгр | O: Ср. путь Выгр-Погр | P: Комментарий
  Строка 3: E: Въезд | F: Выезд | G: Стоянка | H: Въезд | I: Выезд | J: Стоянка
             (остальные merge с строкой 2)

Данные: ОДНА СТРОКА = ОДИН РЕЙС
  Для каждого ТС:
    → Столбцы A,C,D,K,L,M,N,O — ВЕРТИКАЛЬНО ОБЪЕДИНЕНЫ (одинаковы для всех рейсов)
    → Столбцы E-J — уникальны для каждого рейса (въезд/выезд/стоянка погрузки и выгрузки)
    → Столбец B — госномер (не объединён, повторяется)
    → Столбец P — комментарий (не объединён)
```

### Цветовая схема

| Элемент | Цвет фона | Цвет текста | Шрифт |
|---------|-----------|-------------|-------|
| Заголовок дата+смена | Серый #D9D9D9 | Чёрный (theme:1) | Calibri, Bold, sz12 |
| Заголовки столбцов (строки 2-3) | Ярко-синий #0070C0 | Белый #FFFFFF | Calibri, Bold, sz11 |
| Данные | Без фона | Чёрный | Calibri, sz11 |

### Особенности

- **216 объединённых ячеек** — массивное вертикальное объединение по ТС
- Привязка: конкретная дата + конкретная смена (не период)
- Рамки: medium+thin
- Ширина столбцов: A=5.7, B=15.7, C-K=8.7-10.7, L-O=10.7, P=20.7
- Время — Excel serial dates (отображается как ЧЧ:ММ)
- Столбец P (Комментарий) — пустой, для ручного заполнения

### Данные из БД

Источник: `dump_trucks.trips` + `dump_trucks.shift_records` + `dump_trucks.zone_events`

| Столбец шаблона | Поле БД |
|----------------|---------|
| ГосНомер | `shift_records.vehicle_name` |
| Кол-во рейсов | COUNT trips по shift_record_id |
| Начало смены | MIN(`zone_events.entered_at`) или shift start time |
| Погрузка въезд | `trips.loading_entered_at` |
| Погрузка выезд | `trips.loading_exited_at` |
| Погрузка стоянка | `trips.loading_duration` |
| Выгрузка въезд | `trips.unloading_entered_at` |
| Выгрузка выезд | `trips.unloading_exited_at` |
| Выгрузка стоянка | `trips.unloading_duration` |
| Конец смены | MAX timestamp |
| Ср. стоянка П | AVG(`loading_duration`) по ТС |
| Ср. стоянка В | AVG(`unloading_duration`) по ТС |
| Ср. путь П→В | AVG(`travel_to_unload`) по ТС |
| Ср. путь В→П | AVG(`travel_to_load`) по ТС |
| Комментарий | Нет в БД (пустое поле) |

Можно добавить: название зоны погрузки/выгрузки (`trips.loading_zone`, `trips.unloading_zone`), объект (`shift_records.object_name`).

## Принятые решения и открытые вопросы

### Решения

1. **Гибридный подход к шаблонам** — базовые шаблоны + Claude может модифицировать (добавить/убрать столбцы)
2. **Шаблонные функции вместо сырого generateXlsx:**
   - `generateKipReport(period, filters, columns?)` → лист 1
   - `generateDumpTruckSummary(period, filters, columns?)` → лист 2
   - `generateTripDetail(date, shift, filters?)` → лист 3
   - `generateCustomTable(sheets[])` → свободная таблица
3. **Цветовая схема фиксирована** в шаблоне, Claude не меняет цвета
4. **Группировка управляемая** — Claude может менять по запросу (по объекту / по подразделению / по типу)
5. **Превью обязательно** — перед генерацией Claude показывает текстовый план таблицы

### Открытые вопросы (нужно решить)

1. **Merge vs чередование цвета** — вертикальный merge (216 ячеек на листе 3) красиво, но сложно. Альтернатива: повторять значения + чередовать цвет фона для группировки по ТС
2. **Объединять ли две таблицы самосвалов** (лист 2) в одну или оставить раздельно?
3. **Расход топлива и пробег** — добавлять в КИП-отчёт или делать отдельный отчёт?
4. **Двойная нумерация** (глобальный № + внутри подразделения) — убирать или оставлять?
5. **Пустые столбцы A, B** (отступы для печати) — убирать?
6. **Столбец Комментарий** (лист 3) — оставлять пустым или убрать?

### Что фиксировано (из шаблона, Claude НЕ меняет):
- Цветовая схема
- Шрифт и размеры
- Логика merge
- Порядок: заголовок → шапка → группы → данные → итоги

### Что Claude МОЖЕТ менять по запросу:
- Какие столбцы включить/исключить
- Период и фильтры (объект, тип ТС, подразделение)
- Группировку
- Одна смена или обе
- Один лист или несколько

## Поведение AI-агента (system prompt)

### Обязательные параметры (спросить если не указаны):
- Период (за какие даты)
- Тип отчёта (КИП / самосвалы сводка / рейсы детально)

### Необязательные (дефолты):
- Объект → все объекты
- Тип ТС → все типы
- Смены → объединённые (обе смены вместе)
- Группировка → по дате

### Правила интерпретации:
- "эта неделя" → текущая (пн — сегодня)
- "прошлая/та неделя" → пн-вс прошедшей недели
- "за март" → 01.03 — 31.03

### Превью перед генерацией:
Claude ВСЕГДА показывает план таблицы перед генерацией:
```
📋 Отчёт: Рейсы самосвалов
Период: 17.03 — 23.03.2026
Объект: Тобольск
Смены: общий

| Дата | Госномер | Рейсов | Время работы | КИП% |
| ... |
| ИТОГО | | сумма | сумма | среднее |

Листы: 1 | Строк: ~45
Генерирую?
```
