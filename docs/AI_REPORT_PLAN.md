## 0. Принятые решения (ресерч)

| Компонент | Выбор | Альтернативы (отвергнуты) | Почему |
|-----------|-------|--------------------------|--------|
| AI Framework | **Vercel AI SDK v6** (`ai` + `@ai-sdk/anthropic`) | Anthropic SDK напрямую, LangChain.js, Mastra, Claude Agent SDK | Встроенный tool-calling loop, стриминг в Express, `useChat` хук, 23K stars, 9.8M npm/week |
| Модель | **Claude Haiku 4.5** | GPT-4o-mini, Gemini Flash | Дешевле, нативный tool-use, prompt caching (90% экономия), $0.02-0.05 за отчёт |
| XLSX генерация | **ExcelJS** | SheetJS (бесплатная версия без стилей), xlsx-populate (мёртвый) | Полное форматирование (шрифты, цвета, рамки, авто-фильтры), стриминг, MIT, 5.5M npm/week |
| Chat UI | **`useChat` (@ai-sdk/react) + shadcn/ui** | assistant-ui, @chatscope, Deep Chat | shadcn уже в проекте, useChat убирает 80% бойлерплейта, нет лишних зависимостей |
| Стриминг | **SSE** (встроен в AI SDK) | WebSocket, polling | Стандарт индустрии для LLM, `pipeUIMessageStreamToResponse()` одной строкой |
| RAG | **Не нужен** | LlamaIndex, Pinecone | Данные структурированы в БД → чистый tool-use |
| Conversation memory | **In-memory через useChat** | PostgreSQL, Redis | Отчёты — короткие диалоги (3-10 реплик), персистентность не нужна |

### Стоимость
- **Разработка:** $0 (все библиотеки бесплатные, open-source)
- **Эксплуатация:** только Anthropic API — ~$12-30/мес при 20 отчётах/день
- **Prompt caching:** system prompt + tool definitions кешируются → чтение кеша $0.10/M vs $1/M (90% экономия)

## 4. Зависимости между фазами

```
Фаза 0 (подготовка)
  ├─→ 0.1 (аудит вычислений) ─→ Фаза 2 (tools зависят от данных в БД)
  ├─→ 0.2 (шаблоны)          ─→ Фаза 5 (XLSX шаблоны)
  └─→ 0.3 (описание данных)  ─→ Фаза 3 (system prompt)

Фаза 1 (скелет) ─→ Фаза 2 (tools) ─→ Фаза 3 (chat handler)
                                            │
Фаза 4 (фронтенд) ←────────────────────────┘

Фаза 5 (доводка) — после всего
```

**Можно параллелить:**
- Фаза 1 + Фаза 0 (подготовка данных пока пишем скелет)
- Фаза 4 (фронтенд) можно начать после Фазы 1 (mock данные)

## 5. Риски и открытые вопросы

| Вопрос | Статус | Решение |
|--------|--------|---------|
| Хардкод 24т/15м³ за рейс самосвала — откуда брать реальные данные? | Открыт | Нужен источник (TIS API? ручной ввод?) |
| `effectiveNorm()` — где хранить нормы рейсов? | Открыт | Сейчас на фронте, нужно в БД |
| Реестр ТС (госномер → тип, модель, СМУ) — единый источник? | Открыт | Проверить есть ли таблица или брать из TIS |
| Prompt caching требует >4096 токенов в system prompt | Низкий риск | System prompt + tool definitions легко превысят порог |
| Rate limit Anthropic API при массовых отчётах | Низкий риск | 10-20 отчётов/день — далеко от лимитов |
