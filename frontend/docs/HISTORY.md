# Frontend — История и Возможности

## Что реализовано

- React Router v7 (BrowserRouter) с 5 основными роутами + fallback
- TopNavBar с навигацией: 4 основных раздела + «Состояние ТС» + «Гео» + переключатель темы
- Темизация light/dark через `next-themes` с CSS-переменными (2 полных набора)
- Кастомные SVG-иконки ТС: DumpTruckIcon, SemiTruckIcon, HeavyMachineryIcon

### Главная страница (`/`)

- 3-колоночный дашборд (TyagachiVehicleBlock | ReportsColumn | DstMonitoring)
- `TyagachiVehicleBlock` на главной показывает реальные данные тягачей
- `ReportsColumn` поддерживает создание legacy-отчётов и историю отчётов
- `DstMonitoring` — статичная заглушка с анимированными KPI-карточками

### КИП техники (`/kip`)

- Встроен через `<iframe>` с передачей темы через query-параметр
- `hostname` динамический — работает при обращении с других устройств в сети

### Тягачи (`/tyagachi/*`)

- Полный React фронтенд (заменяет HTML-генератор для интерактивной части)
- Dashboard: SyncPanel (1д/3д/1н/1м) + VehicleOverview с таймлайном заявок по ПЛ
- Синхронизация с прогресс-баром и поллингом статуса
- ReportView: 3-колоночный layout (список ПЛ | карта Leaflet | факт-панель)
- Трек на карте: синий Polyline + красные CircleMarker (стоянки) + Tooltip
- Автовыбор первой машины с треком, авто-fitBounds
- VehicleOverview: expandable строки машин с таймлайном ПЛ (buildStackedSegments)
- Типизация TypeScript для всех API-ответов (types.ts)

### Самосвалы (`/samosvaly`)

- Полный React-компонент DumpTrucksPage (портирован из samosvaly-v6.html)
- Собственные CSS-стили (samosvaly.css)
- Группировка по объектам (городам)
- API: объекты, заявки, смены (shift-records), детали смены (trips + zone events), ремонты

### Состояние ТС (`/vehicle-status`)

- Таблица 8 колонок с фильтрацией по статусу и категории
- Ручная синхронизация из Google Drive (.xlsx) с поллингом
- Цветовая подсветка строк в ремонте
- 8 категорий техники (соответствуют вкладкам Excel)

## Что можно получить из текущей архитектуры

**Без изменений архитектуры:**

- Добавить новый раздел: создать `features/новый/`, роут в App.tsx, пункт в TopNavBar.tsx, proxy в vite.config.ts — стандартный шаблон уже отработан
- Добавить фильтрацию в TyagachiDashboard по типу машины (API уже поддерживает `?days=N`)
- Расширить VehicleStatusPage: экспорт таблицы, сортировка по столбцам
- Показывать трек нескольких машин одновременно в TyagachiReportView (карта уже готова)
- Добавить уведомления через `sonner` (зависимость уже установлена)
- Добавить компоненты из shadcn/ui — все примитивы уже установлены (accordion, dialog, tabs и т.д.)

**Со средними изменениями:**

- Перевести KIP техники с iframe на React-компоненты (API kip/server доступен через `/api/kip`)
- Добавить живой мониторинг (WebSocket или SSE) — нужно добавить поддержку на бэкенде
- Сделать DstMonitoring живым (подключить к реальному API)
- Добавить экспорт данных из DumpTrucksPage (CSV/Excel)
- Добавить роут `/geo` с Leaflet-картой геозон (geo-admin уже работает отдельно на :3003)

**С существенными изменениями:**

- Полный переезд тягачей с legacy HTML-отчётов на React (ReportView уже есть, нужно перенести всю логику html_generator_v2.py)
- Единый state management (сейчас каждый раздел — изолированный state)
- Мобильная адаптация (сейчас layout ориентирован на desktop, h-screen с overflow-hidden)

## Что ограничено архитектурой

- **КИП через iframe**: невозможна глубокая интеграция (shared state, единая навигация внутри KIP, передача данных). Только тема передаётся через query-параметр.
- **h-screen + overflow-hidden**: вся страница фиксированной высоты, нет скролла на уровне body. Разделы должны сами управлять overflow внутри. Это ограничивает мобильную адаптацию.
- **samosvaly.css**: CSS-стили самосвалов изолированы в собственном файле и частично конфликтуют с Tailwind (используют нестандартные классы `sv-*`). Требует рефакторинга при изменении дизайна.
- **DstMonitoring**: полностью статичный компонент, хардкоженные данные. Не подключён ни к какому API.
- **Нет авторизации**: приложение полностью открыто, нет аутентификации.
- **Нет .env для фронтенда**: BASE URL бэкендов хардкодятся как `localhost:PORT` через Vite proxy. При деплое нужна конфигурация nginx.
- **Legacy-отчёты тягачей**: открываются в новой вкладке как HTML (путь `/api/tyagachi/reports/:id/v2`), не интегрированы в React-интерфейс.
